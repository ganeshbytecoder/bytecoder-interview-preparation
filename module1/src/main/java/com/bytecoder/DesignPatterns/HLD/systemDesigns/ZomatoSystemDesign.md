## Design Online Delivery System

### Functional requirements 
- user should be order food from a restaurant 
- will search by food item category name or restaurant name 
- user can select list of items and place a order -> apply coupon ->
  - select address and payment - track order after order is placed 
- home page will contain list of categories and personalised suggestions nearby that location 
- Live tracking the order
- Cancel the order 
- review and feedback, coupons , 
- Book a table for Dine-in or order for take away or delivery 
- notifications 
- customer support 
- payments
- aids
- order history

- scalable , secured and fault -torrent system,  low latency
- JWT token ROLE based based authentication
